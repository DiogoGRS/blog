Manipulação de Strings em JavaScript String Manipulation in JavaScript

trim(), replace() com regex, normalize() e Intl.Collator: as ferramentas essenciais para trabalhar com strings. trim(), replace() with regex, normalize() and Intl.Collator: the essential toolkit for working with strings.

trim()

O método trim() remove espaços em branco e quebras de linha (\n) do início e do fim de uma string, retornando uma nova string — sem modificar a original. 

"  hello world  ".trim()
// → "hello world"

"  linha 1\n".trim()
// → "linha 1"

Para remover apenas de um lado: trimStart() limpa o início, trimEnd() limpa o fim.

replace() e Expressões Regulares

O método replace() é a ferramenta mais versátil para manipulação de strings. Quando combinado com expressões regulares (regex), permite encontrar e substituir padrões complexos.

A sintaxe de um regex é: /padrão/flags

Padrões

// | (OR) — match com "joão" ou "maria"
/joão|maria/

// [A-Z] (intervalo) — qualquer caractere entre A e Z
/[A-Z]/

// \D (atalho) — qualquer coisa exceto dígito
// combinado com replace("") remove tudo que não é número
/\D/g

// () (grupos de captura) — torna partes do padrão acessíveis
// como parâmetros na função de substituição
/([a-z]+)|([0-9]+)/gi

Flags

// g — todas as ocorrências (sem ela, só o primeiro match é substituído)
// i — case-insensitive
// m — multi-line (^ e $ consideram quebras de linha)

Funções de Substituição

O segundo argumento do replace() pode ser uma função em vez de uma string. Essa função recebe até 5 parâmetros: 

string.replace(regex, (match, p1, p2, offset, originalString) => {
  // match          → a substring completa que deu match
  // p1, p2...      → conteúdo dos grupos de captura (parênteses no regex)
  // offset         → posição onde o match começa na string original
  // originalString → a string completa sendo processada
  return "novo valor"
})

Exemplo prático — converte letras para maiúsculo e números para soma dos dígitos:

const result = str.replace(/([a-z]+)|([0-9]+)/gi, (match, p1, p2) => {
  if (p1) return p1.toUpperCase()
  if (p2) {
    const digits = Array.from(p2)
    return digits.reduce((acc, d) => acc + Number(d), 0).toString()
  }
})

normalize("NFD") — Removendo Acentos

O método normalize("NFD") decompõe caracteres acentuados em sua forma base + acento separado. Isso permite remover os acentos com replace(): 

const semAcento = str
  .normalize("NFD")
  .replace(/[\u0300-\u036f]/g, "")

// "Ação"      → "Acao"
// "São Paulo" → "Sao Paulo"

O browser normaliza automaticamente ao exibir strings decompostas — visualmente você ainda veria a frase original. Para enxergar os caracteres separados de fato, use Array.from(str.normalize("NFD")).

Intl.Collator — Comparação sem Modificar a String

Para comparar strings de forma locale-aware sem alterar os valores originais, use Intl.Collator. É a alternativa mais limpa ao localeCompare quando você precisa comparar muitas strings (o Collator é instanciado uma vez e reutilizado, o que é mais eficiente). 

const collator = new Intl.Collator('pt', { sensitivity: 'base' })

collator.compare("cafe", "café")   // → 0 (iguais para fins de comparação)
collator.compare("a", "b")         // → número negativo
collator.compare("b", "a")         // → número positivo

Com sensitivity: 'base', acentos e diferenças entre maiúsculas e minúsculas são ignorados na comparação — útil para buscas e ordenações que precisam ser tolerantes a variações.

trim()

The trim() method removes whitespace and line breaks (\n) from the beginning and end of a string, returning a new string — without modifying the original. 

"  hello world  ".trim()
// → "hello world"

"  line 1\n".trim()
// → "line 1"

To remove from one side only: trimStart() cleans the beginning, trimEnd() cleans the end.

replace() and Regular Expressions

The replace() method is the most versatile string manipulation tool. When combined with regular expressions (regex), it allows finding and replacing complex patterns.

Regex syntax: /pattern/flags

Patterns

// | (OR) — matches "john" or "mary"
/john|mary/

// [A-Z] (range) — any character between A and Z
/[A-Z]/

// \D (shorthand) — anything except a digit
// combined with replace("") removes everything that isn't a number
/\D/g

// () (capture groups) — makes parts of the pattern accessible
// as parameters in the replacement function
/([a-z]+)|([0-9]+)/gi

Flags

// g — all occurrences (without it, only the first match is replaced)
// i — case-insensitive
// m — multi-line (^ and $ consider line breaks)

Replacement Functions

The second argument of replace() can be a function instead of a string. This function receives up to 5 parameters: 

string.replace(regex, (match, p1, p2, offset, originalString) => {
  // match          → the full substring that matched
  // p1, p2...      → content of capture groups (parentheses in the regex)
  // offset         → position where the match begins in the original string
  // originalString → the full string being processed
  return "new value"
})

Practical example — converts letters to uppercase and numbers to digit sum:

const result = str.replace(/([a-z]+)|([0-9]+)/gi, (match, p1, p2) => {
  if (p1) return p1.toUpperCase()
  if (p2) {
    const digits = Array.from(p2)
    return digits.reduce((acc, d) => acc + Number(d), 0).toString()
  }
})

normalize("NFD") — Removing Accents

The normalize("NFD") method decomposes accented characters into their base form + separate accent mark. This allows removing accents with replace(): 

const withoutAccents = str
  .normalize("NFD")
  .replace(/[\u0300-\u036f]/g, "")

// "Ação"      → "Acao"
// "São Paulo" → "Sao Paulo"

The browser automatically normalizes decomposed strings on display — you'd still see the original phrase visually. To actually see the separated characters, use Array.from(str.normalize("NFD")).

Intl.Collator — Comparison Without Modifying the String

To compare strings in a locale-aware way without altering the original values, use Intl.Collator. It's the cleanest alternative to localeCompare when comparing many strings (the Collator is instantiated once and reused, which is more efficient). 

const collator = new Intl.Collator('en', { sensitivity: 'base' })

collator.compare("cafe", "café")   // → 0 (equal for comparison purposes)
collator.compare("a", "b")         // → negative number
collator.compare("b", "a")         // → positive number

With sensitivity: 'base', accents and case differences are ignored in the comparison — useful for searches and sorts that need to be tolerant of variations.
