Ordenação de Arrays em JavaScript Array Sorting in JavaScript

Como sort() e toSorted() funcionam por baixo dos panos — e por que 10 vem antes do 2. How sort() and toSorted() work under the hood — and why 10 comes before 2.

sort() e toSorted()

O JavaScript tem duas funções nativas para ordenar arrays: sort() e toSorted(). O comportamento de ambas é idêntico, com uma diferença importante: toSorted() cria uma cópia do array original antes de ordenar, preservando o original intacto. sort() modifica o array diretamente (in-place).

Se você precisar preservar o array original, prefira toSorted() ou crie uma cópia manualmente: [...arr].sort().

Ordenação Lexicográfica

Quando chamada sem argumentos, sort() ordena os elementos de forma lexicográfica — trata tudo como string e compara caractere por caractere, usando a tabela UTF-16. Isso significa que um array de números não será ordenado numericamente: 

[10, 2, 1, 21, 3].sort()
// → [1, 10, 2, 21, 3]

Por que 10 aparece antes do 2?

Porque "10" começa com "1", e "2" começa com "2". O sort avalia o primeiro caractere como critério principal; os seguintes servem apenas como desempate. O elemento arr[0] define a ordem — arr[1], arr[2]... só entram em cena quando há empate.

UTF-16 e o Problema com o Português

A comparação UTF-16 converte cada caractere em sua Code Unit e compara esses números. Maiúsculas e minúsculas têm Code Units diferentes, e acentos possuem valores muito maiores que as letras básicas: 'Z' → 90 · 'a' → 97 · 'á' → 225 · 'ç' → 231

Isso quebra a ordenação alfabética em português — acentos acabam sempre indo para o final, pois seus Code Units são maiores que qualquer letra sem acento: 

["Zebra", "Árvore", "Caça"].sort()
// → ["Caça", "Zebra", "Árvore"]
// Esperado: ["Árvore", "Caça", "Zebra"]

A solução é usar localeCompare, que respeita as regras do idioma:

const animais = ["Zebra", "Árvore", "Caça"]

animais.sort((a, b) => a.localeCompare(b))
// → ["Árvore", "Caça", "Zebra"]

Funções de Comparação

Para garantir a ordenação correta de números, passe uma função de comparação. O sort() interpreta o retorno dela da seguinte forma: resultado negativo coloca a antes de b, positivo coloca b antes de a, e zero mantém a ordem atual. 

const numeros = [10, 2, 1, 21, 3]

// Crescente
numeros.sort((a, b) => a - b)
// → [1, 2, 3, 10, 21]

// Decrescente
numeros.sort((a, b) => b - a)
// → [21, 10, 3, 2, 1]

Qualquer função que retorne um número segue essa convenção — a subtração é só o caso mais comum. Internamente, o sort() usa o algoritmo Timsort, um híbrido entre Merge Sort e Insertion Sort, eficiente para dados parcialmente ordenados.

A função passada ao sort() deve sempre retornar um número. Qualquer outra lógica funciona, desde que respeite: negativo, zero ou positivo.

sort( ) and toSorted( )

JavaScript provides two native functions for sorting arrays: sort() and toSorted(). Both behave identically, with one key difference: toSorted() creates a copy of the original array before sorting, leaving the original unchanged. sort() modifies the array directly (in-place).

To preserve the original array, prefer toSorted() or create a copy manually: [...arr].sort().

Lexicographic Ordering

Without arguments, sort() orders elements lexicographically — it treats everything as a string and compares character by character using the UTF-16 table. This means a number array won't be sorted numerically: 

[10, 2, 1, 21, 3].sort()
// → [1, 10, 2, 21, 3]

Why does 10 come before 2?

Because "10" starts with "1", and "2" starts with "2". sort() evaluates the first character as the main criterion; subsequent characters are only used as tiebreakers. arr[0] defines the order — arr[1], arr[2]... only come into play on a tie.

UTF-16 and the Comparison Mechanism

UTF-16 comparison converts each character to its Code Unit and compares those numbers. Uppercase and lowercase letters have different Code Units, and accented characters have much larger values than basic letters: 'Z' → 90 · 'a' → 97 · 'á' → 225 · 'ç' → 231

This can cause unexpected results when sorting strings with accented characters:

["Zebra", "Árvore", "Caça"].sort()
// → ["Caça", "Zebra", "Árvore"]
// Expected: ["Árvore", "Caça", "Zebra"]

The solution is localeCompare, which respects language-specific rules:

const animals = ["Zebra", "Árvore", "Caça"]

animals.sort((a, b) => a.localeCompare(b))
// → ["Árvore", "Caça", "Zebra"]

Comparison Functions

To ensure correct numeric sorting, pass a comparison function. sort() interprets its return value as follows: a negative result places a before b, positive places b before a, and zero keeps the current order. 

const numbers = [10, 2, 1, 21, 3]

// Ascending
numbers.sort((a, b) => a - b)
// → [1, 2, 3, 10, 21]

// Descending
numbers.sort((a, b) => b - a)
// → [21, 10, 3, 2, 1]

Any function returning a number follows this convention — subtraction is just the most common case. Internally, sort() uses the Timsort algorithm, a hybrid of Merge Sort and Insertion Sort, efficient for partially ordered data.

The function passed to sort() must always return a number. Any logic works, as long as it returns: negative, zero, or positive.
